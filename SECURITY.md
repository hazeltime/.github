# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| < Latest | :x:               |

Only the most recent release of Haz3l receives security updates. Please ensure you are running the latest version before reporting an issue.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly. **Do not open a public issue.**

- **Preferred:** Use [GitHub Private Vulnerability Reporting](https://github.com/hazeltime/.github/security/advisories/new) to submit a report directly.
- **Alternative:** Email the security contact at **berrat** (GitHub: [@berrat](https://github.com/berrat)).

### What to include

- A description of the vulnerability and its potential impact.
- Steps to reproduce or a proof-of-concept, if possible.
- The affected version(s) and component(s).

### Response timeline

- **Acknowledgement:** within 3 business days.
- **Initial assessment:** within 7 business days.
- **Resolution or mitigation:** as soon as practically possible, depending on severity.

We appreciate your help in keeping Hazel and its users safe.
